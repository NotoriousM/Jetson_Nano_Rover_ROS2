## 9.1 Прямолинейная траектория

**Нода:** `straight_trajectory_node`

```bash
# Запуск
ros2 launch rover_nodes trajectory_control.launch.py

# Вызов сервиса
ros2 service call /start_straight_trajectory \
  rover_interfaces/srv/StartStraightTrajectory \
  '{distance: 2.0, speed: 0.5}'    # 2 метра вперёд
  
ros2 service call /start_straight_trajectory \
  rover_interfaces/srv/StartStraightTrajectory \
  '{distance: -1.0, speed: 0.3}'   # 1 метр назад
```

**Мониторинг:**
```bash
ros2 topic echo /trajectory/status
```

**Профиль скорости:**
```
Скорость
  v_max ─────────────────────────────────────────────────────────
                                                 \              |
  v_brake ─────────────────────────────────────── \─────────────
                                                              |
       0 ─────────────────────────────────────────────────────●
       
       Старт         Движение           Зона торможения  Стоп
                                        ← brake_dist=0.3 →
```

**Параметры:**
```yaml
brake_distance:   0.3    # начинать тормозить за 30 см до цели
brake_speed:      0.1    # минимальная скорость при торможении
finish_tolerance: 0.05   # допуск окончания 5 см
max_speed:        1.5    # ограничение скорости
```

## 9.3 Математика кривой Лиссажу

Траектория лемнискаты реализована через кривую Лиссажу с соотношением частот 1:2:

```
Параметрические уравнения:
  x(s) = a · sin(s)          (вдоль +X — вперёд)
  y(s) = a · sin(2s) / 2     (вдоль +Y — влево)
  
  a = scale параметр (м), s ∈ [π/4, π/4 + 2π·loops]

Производные по параметру s:
  dx/ds = a · cos(s)
  dy/ds = a · cos(2s)
  d²x/ds² = −a · sin(s)
  d²y/ds² = −2a · sin(2s)

Скорость параметризации:
  |v(s)| = √(dx² + dy²) = a · √(cos²(s) + cos²(2s))

Кривизна (формула Френе):
  κ(s) = (dx · d²y − dy · d²x) / |v(s)|³

Угол руля (обратная кинематика):
  δ(s) = −atan(L · κ(s))    [знак минус из ориентации base_footprint]
  δ(s) = clip(δ(s), ±δ_max)
```

**Ключевые свойства:**

| Параметр s | Свойство |
|-----------|---------|
| `s = π/4` | `dy/ds = 0` → старт в направлении +X (вперёд) ✓ |
| `s = 0, π` | `κ = 0` → ровер проходит пересечение прямо ✓ |
| `s = π/2` | `κ = max` → пик правой петли |
| `s = 3π/2` | `κ = min` → пик левой петли |

**Ограничение масштаба:**
```
a_min = 4·a_distance / tan(δ_max)
      = 4 · 0.4035 / tan(30°)
      = 4 · 0.4035 / 0.577
      ≈ 2.80 м

При scale < a_min угол руля клэмпится → деформация петель.
Рекомендуется scale ≥ 3.0 м.
```

## 9.4 Arc-length параметризация

**Проблема:** если продвигать параметр `s` с постоянной скоростью, физическая скорость ровера будет меняться (зависит от `|v(s)|`).

**Решение:** arc-length параметризация — продвигать `s` пропорционально `1/|v(s)|`:

```python
def advance_parameter(s, v_robot, dt):
    """Продвинуть параметр s так, чтобы физическая скорость была v_robot = const."""
    dx = a * cos(s)
    dy = a * cos(2*s)
    spd = sqrt(dx*dx + dy*dy)         # скорость параметризации
    
    ds = v_robot * dt / max(spd, 1e-6)  # шаг по параметру
    return s + ds
```

Это обеспечивает: робот всегда едет со скоростью `v_robot` м/с, независимо от кривизны.

## 9.5 PID-регулятор следования

Для следования по траектории реализован PID по поперечной ошибке:

```
e_y   = -(x-x_r)·sin(ψ_r) + (y-y_r)·cos(ψ_r)   # поперечная ошибка
e_ψ   = atan2(sin(ψ-ψ_r), cos(ψ-ψ_r))           # ошибка курса

δ_fb  = -(Kp·e_y + Ki·∫e_y + Kd·ė_y) + Kψ·e_ψ  # обратная связь
δ_ff  = atan(L · κ_ref)                           # фидфорвард по кривизне
δ_cmd = clip(δ_fb + δ_ff, ±δ_max)
```

**Anti-windup:** если `|δ_cmd| > δ_max`, интегратор не накапливается.

**Рекомендованные параметры** (v = 0.4 м/с):

| Параметр | Значение |
|----------|---------|
| `Kp` | 1.5 |
| `Ki` | 0.05 |
| `Kd` | 0.3 |
| `Kψ` | 0.8 |

## 9.6 LQR-регулятор следования

Оптимальный регулятор минимизирует:
```
J = ∫(zᵀQz + uᵀRu) dt,   z = [e_y, e_ψ, ∫e_y]ᵀ
```

**Синтез** через решение CARE (`scipy.linalg.solve_continuous_are`):
```
AᵀP + PA − PBR⁻¹BᵀP + Q = 0
K = R⁻¹BᵀP
```

**Управление:**
```
u_fb  = −K · z
δ_cmd = clip(u_fb + atan(L·κ_ref), ±δ_max)
```

**Планирование усиления:** матрица K вычисляется для каждой скорости из набора {0.2, 0.3, 0.4, 0.5, 0.7, 1.0} м/с и интерполируется онлайн.

**Запуск:**
```bash
ros2 launch rover_description figure_eight_lqr.launch.py \
  scale:=3.0 speed:=0.4 q_ey:=10.0 q_epsi:=5.0 r_delta:=0.1
```
